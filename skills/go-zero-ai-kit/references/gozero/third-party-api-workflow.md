# 第三方 API 封装流程

## 适用场景
- 在 `pkg/provider/` 下新增第三方 API 客户端
- 对已有第三方 API 封装做重构、补接口、补测试、补注释
- 统一不同第三方系统的接入风格与 review 标准

## 核心规则
- 先读同目录已有实现，优先复用现有目录结构和文件拆分方式。
- 对外方法名优先使用我方业务语义，不直接照搬第三方平台术语。
- 字段名与 JSON tag 需要和第三方协议保持可映射，避免协议层失真。
- 注释优先按第三方文档的真实语义补齐，不写空泛注释。
- 一个接口一个文件，`client.go` 只放客户端、配置、公共 headers 和接口定义。
- 不做 `req == nil` 判断，除非协议层显式允许空请求且已有统一模式。
- 先封装协议层，再考虑是否接入业务 logic，不要边猜边接。
