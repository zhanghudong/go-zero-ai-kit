# 数据库规范

适用场景：
- 新增或调整表结构约定
- 在 logic/model 中设计查询方式
- 需要处理主从一致性或幂等写入

核心规则：
- 表名使用小写下划线。
- 主键统一使用 snowflake 或业务约定主键策略。
- 软删除字段使用 `deleted_at`。
- 索引命名统一为 `idx_<table>_<columns>`。
- 刚写后读且要求强一致时，必须查询主库，避免读写分离延迟带来脏读或未读到。
- 需要主库查询时，在 model 中增加明确的方法，例如 `FindOneByMaster`，不要在 logic 层拼接 SQL。
- 主库查询 SQL 统一使用 `/*FORCE_MASTER*/` 注释前缀，方法命名统一使用 `ByMaster` 后缀。

禁止事项：
- 在 logic 层直接拼接一段临时 SQL 绕过 model。
- 对强一致读取场景默认走从库。
- 主库查询方法命名和 SQL 标记不统一。
