# Logic 业务模式库（严格版）

适用场景：
- 已明确要在 logic 中实现某类通用业务流程
- 需要快速选择一个稳定的实现骨架
- 想减少重复造轮子，但不希望把模式文档写成风格规范

使用方式：
- 本文档只提供模式骨架和注意点，不重复定义通用风格规则。
- 具体日志、错误处理、幂等工具、字段命名仍以 `logic-style.md` 为准。
- 实现完成后仍需按 `logic-checklist.md` 自检。

## 模式1：查询 + 返回
- 适用于详情查询、单条读取、校验后返回。
- 步骤1：查询数据
- 步骤2：判断不存在/非法状态
- 步骤3：拼装响应
- 注意：分页默认值来自 .api；`pageSize` 只做上限裁剪

## 模式2：状态迁移
- 适用于审核、启停、发布、关闭等状态流转场景。
- 步骤1：查询当前状态
- 步骤2：校验状态可迁移
- 步骤3：更新状态（注明原因）
- 步骤4：记录审计/日志

## 模式3：创建/写入
- 适用于创建主实体、落库并返回主键的场景。
- 步骤1：检查幂等（如有业务唯一键）
- 步骤2：写入主表
- 步骤3：写入关联表/日志
- 步骤4：返回主键

## 模式4：外部调用
- 适用于请求第三方、RPC 或其他内部服务。
- 步骤1：组装请求
- 步骤2：调用外部服务
- 步骤3：处理异常/超时
- 步骤4：落库或返回

## 模式5：更新（按字段选择性更新）
- 适用于 patch 风格更新或“只改传入字段”的更新场景。
- 步骤1：查询当前状态
- 步骤2：对非空字段构建更新内容
- 步骤3：执行更新
- 步骤4：返回结果

## 模式6：列表分页查询
- 适用于带筛选条件的列表接口。
- 步骤1：构建 `RowBuilder` 与 `CountBuilder("*")`
- 步骤2：按条件追加 where（字段统一加反引号）
- 步骤3：先查总数，再查分页数据

## 模式7：异步任务（主流程快速返回）
- 适用于主链路完成后，异步发送通知、刷缓存、触发后续动作。
- 步骤1：主流程完成核心落库并返回响应
- 步骤2：复制必要入参，避免共享可变对象
- 步骤3：使用新 context 保留 trace，脱离请求 cancel
- 步骤4：设置任务超时并使用 `threading.GoSafeCtx` 执行
- 步骤5：异步失败只记录错误日志，不影响主流程结果
- 注意：异步任务内禁止依赖已取消的请求上下文

## 模式8：分布式锁（按业务键串行）
- 适用于高并发下按业务键串行化处理，例如重复提交、防并发状态流转。
- 步骤1：按业务主键构建 lock key（如 `orderId`）
- 步骤2：优先使用项目统一封装的分布式锁组件，并设置过期时间/重试参数
- 步骤3：优先使用 `DoWithLock` 包裹临界区；仅在确有需要时手动 `Acquire/Release`
- 步骤4：获取锁失败时区分系统异常与锁竞争
- 步骤5：临界区内处理状态流转、幂等等并发安全逻辑
- 注意：临界区内状态迁移应使用 CAS；`affected == 0` 视为并发命中并幂等退出
- 注意：锁超时、重试次数、重试间隔必须按具体逻辑耗时和并发特征评估，示例值不可直接视为固定标准
